{% trans "What we do" %}

  • {% trans "Full keyboard navigation for all interactive elements" %}
  • {% trans "Visible focus indicators on every focusable element" %}
  • {% trans "Color contrast ratios of at least 4.5:1 for text" %}
  • {% trans "Touch targets of at least 44×44 pixels on mobile" %}
  • {% trans "Semantic HTML and ARIA labels for screen readers" %}
  • {% trans "Responsive design that works across devices and zoom levels" %}
  • {% trans "Enhanced contrast ratios of at least 7:1 for body text" %}
  • {% trans "No time limits on reading legal information" %}
  • {% trans "Plain-language definitions for legal terms and abbreviations" %}

{% trans "Testing" %}

{% blocktrans %}We combine automated accessibility checks on every code change with manual keyboard and screen reader testing for each new feature. Periodic full-site audits verify conformance against the complete WCAG checklist.{% endblocktrans %}

{% trans "Standards tracking" %}

{% blocktrans %}We review our practices against current W3C accessibility guidelines annually and will update our targets as standards evolve.{% endblocktrans %}

{% trans "Known limitations" %}
