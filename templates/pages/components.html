Litigant Portal

Component Library

Living documentation for Cotton components with Tailwind CSS. These components are available to all templates via <c-componentname> syntax.

We use Django Cotton for components, Alpine.js for interactivity, and Tailwind CSS for styling.

{# ========== BUTTONS ========== #}

Buttons

Accessible button component with multiple variants. WCAG 2.1 AA compliant.

{# Demo #}

Demo

Primary Outline Dark Ghost Danger
Small Default Extra Large
Disabled
{# Props #}

Props

variant "primary" | "outline" | "dark" | "ghost" | "danger". Default: "primary"
size "sm" | "" | "xl". Default: "" (normal size)
type "button" | "submit" | "reset". Default: "button"
disabled
(optional) Boolean. Disables the button.
full_width
(optional) Boolean. Makes button full width.
{# Code #}
Show code 
<c-button variant="primary">Primary</c-button>
<c-button variant="outline">Outline</c-button>
<c-button variant="dark">Dark</c-button>
<c-button variant="ghost">Ghost</c-button>
<c-button variant="danger">Danger</c-button>

<c-button variant="primary" size="sm">Small</c-button>
<c-button variant="primary" size="xl">Extra Large</c-button>

<c-button variant="primary" disabled>Disabled</c-button>
{# ========== INPUTS ========== #}

Inputs

Accessible input component with error and success states.

{# Demo #}

Demo

{# Props #}

Props

type "text" | "email" | "password" | "tel" | etc. Default: "text"
name
(optional) Form field name
placeholder
(optional) Placeholder text
error
(optional) Boolean. Shows error state styling.
success
(optional) Boolean. Shows success state styling.
disabled
(optional) Boolean. Disables the input.
{# Code #}
Show code 
<c-input placeholder="Default input" />
<c-input type="email" placeholder="Email input" />
<c-input placeholder="Error state" error />
<c-input placeholder="Success state" success />
<c-input placeholder="Disabled" disabled />

{# ========== LINKS ========== #}

Links

Accessible link component. Auto-handles external link security with rel="noopener noreferrer".

{# Demo #}

Demo

Default Link Primary Link Secondary Link Unstyled Link
External Link
{# Props #}

Props

href URL for the link
variant "default" | "primary" | "secondary" | "unstyled". Default: "default"
target
(optional) "_blank" to open in new tab
external_icon
(optional) Boolean. Shows external link icon (use with target="_blank")
{# Code #}
Show code 
<c-link href="#" variant="default">Default</c-link>
<c-link href="#" variant="primary">Primary</c-link>
<c-link href="#" variant="secondary">Secondary</c-link>
<c-link href="#" variant="unstyled">Unstyled</c-link>

<c-link href="https://..." target="_blank" external_icon>External</c-link>
