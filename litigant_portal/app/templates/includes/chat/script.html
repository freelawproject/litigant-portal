{% comment %} Chat component — chatApp() Alpine factory + tool render registry. Self-contained. The factory is exposed on `window.chatApp` so any markup that wraps itself in `x-data="chatApp({ agentName: '...' })"` participates in the same conversation state. Markup lives in: - includes/chat/conversations.html (sidebar) - includes/chat/thread.html (center column) Public surface: window.chatApp({ agentName }) → Alpine factory window.registerChatTool(name, handlers) → register custom tool renderers handlers = { renderCall(item, ctx) → body HTML } ctx.findResponse(id) → matching tool_response item, or null Tool rendering: every tool invocation is rendered as a single row driven by the tool_call item. The renderer can look ahead via ctx.findResponse(id) to pull in the result once it arrives — the row's status badge flips from "Working…" to "Done" automatically. tool_response items live in the items array as pure data and are never rendered on their own. Wire protocol — the streaming endpoint speaks NDJSON: one JSON event per line, each event being whatever Agent.stream_run yielded. The thread id for the current request lives in the X-Thread-Id response header so the event stream itself stays unmodified. {% endcomment %}
